## Governance

Phase 4B operates a **vetted-manifest model**. This document records *needs and candidate source categories only*; it asserts no specific URLs, attributions, or licenses as fact.

1. **Per-image human approval.** For each row, a specific candidate image is researched, its provenance and license individually verified, and the user explicitly approves that image **before any binary is committed**. Nothing in this manifest authorises committing an image.
2. **Build-time integrity gate.** Per Phase-4A spec §6 and the content integrity gate, any committed image (`hero`/`figure` `ArchiveImage`) must carry complete, valid metadata — non-empty `alt`, a `credit.source`, a `credit.license`, and accurate `width`/`height` dimensions. A stub or metadata-incomplete image fails `npm run test:content` / `npm run build`; this is intentional and must not be bypassed.
3. **No AI-generated or scraped imagery.** Per Phase-4A spec §6, no AI-synthesised images and no scraped/unverified-provenance images are eligible. Only material from a verified public-domain or appropriately-licensed source (Wikimedia Commons, Library of Congress, or a public museum / manufacturer heritage archive with a confirmed open license) may be proposed for approval.
4. **Graceful fallback is acceptable.** Pages render with the typographic Frontispiece when no image exists. If no candidate clears verification and approval for a given row, the page ships image-free rather than with an unvetted or fabricated asset. Image absence is never a release blocker.
5. **No factual drift via captions.** Alt text and captions must honour Phase-4A §2: no invented dates, model names, market share, financials, or "first/largest" claims, and no brand attribution stated as fact where the page itself frames the topic conceptually.
